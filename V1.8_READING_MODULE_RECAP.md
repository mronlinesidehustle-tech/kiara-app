## v1.8 — Reading Module 🆕 (2026-04-27)

Progressive voice-only reading lesson addressing letter reversals and pronunciation challenges. Single-session module covers **Sight Words** → **Phonics** → **Short Story** with Mrs. Love coaching throughout. Parent can customize word lists and story in dashboard settings.

**Flow:** Intro (auto-speak welcome) → 5 sight words (exact match) → 5 phonics words (spaced letters, same match) → story by sentence (≥70% word overlap = point) → completion with per-phase scores.

**Defaults:**
- Sight words: the, and, is, are, it, we, go, my, he, she (10-pool, 5/session)
- Phonics: cat, dog, sun, hat, big, red, fun, sit, hop, map (10-pool, 5/session)
- Story: "The cat sat on the mat. It is a big red mat. The cat is happy."

### Data flow (Reading Module)

1. ReadingLesson component loads → useEffect auto-speaks intro welcome
2. Kiara taps "🎤 Read it!" → createSpeechRecognizer() starts listening
3. Speech recognition triggers onResult callback with transcript
4. Captured phase/index/word list check for exact match (sight words) or ≥70% overlap (story)
5. Match → score increments (via ref), spoken praise, advance word
6. No match → hint or reveal + allow retry (max 2x before moving on)
7. All phases complete → advanceSentence() calls saveProgress() with lessonType: 'reading'
8. Server dedupes and stores in Upstash KV
9. ParentDashboard GETs /api/progress, filters readingSessions, displays per-phase scores

### Critical safety patterns (Reading Module)

- `wordLockedRef` / `sentenceLockedRef`: synchronous boolean guards prevent double-submit during async recognition
- Score refs (sightScoreRef, phonicsScoreRef, storyScoreRef): used in callbacks; state mirrors for render
- Captured variables (capturedPhase, capturedIndex, capturedWordList) in closure at recognition-start prevent stale values in async onResult
- Unmount cleanup useEffect: stops recognizer + stops speaking if back-tapped mid-recognition
- Symmetric normalization: `text.toLowerCase().replace(/[^a-z\s]/g, '').trim()` applied to both transcript and target
