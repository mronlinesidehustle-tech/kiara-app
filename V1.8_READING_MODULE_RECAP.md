## Overview

Added a progressive, voice-only reading lesson for Kiara covering **Sight Words** → **Phonics** → **Short Story** in a single sitting. Mrs. Love (character coach) guides throughout with praise and encouragement, mirroring the math lesson pattern.

---

## What's New

### Home Screen
- New button: **📖 Start Reading Lesson** (below the two math lesson buttons)

### Reading Lesson (New Component)
A single-session lesson with four phases:

1. **Intro** — Mrs. Love auto-speaks welcome message when page loads
2. **Sight Words Phase** — 5 sight words (from default or parent-customized pool), voice recognition matching
3. **Phonics Phase** — 5 CVC words with spaced letters (e.g., `c · a · t`), same matching flow
4. **Short Story Phase** — 3-sentence story shown line-by-line, ≥70% word overlap counts as correct
5. **Completion Screen** — Score breakdown (Sight Words: X/5, Phonics: X/5, Story: X/3), celebration emoji, percentage score with star badge

### Parent Dashboard Enhancements
- **Reading Sessions Tab** — Shows completed reading lessons with per-phase scores and percentages
- **Reading Settings Override** — Parents can customize:
  - Sight words (comma-separated, min 5)
  - Phonics words (comma-separated, min 5)
  - Short story (free text, sentences separated by periods/exclamation/question marks)
  - Save button flashes ✅ when saved
  - Reset to Defaults button clears overrides

### Key Design Patterns

**Score Tracking (Stale-Closure Safe)**
- Dual ref + state pattern: `sightScoreRef` + `setSightScore`
- Refs used in async callbacks (speech recognition), state used for render
- Prevents scores from going stale inside captured closures

**Double-Submit Guard**
- `wordLockedRef` / `sentenceLockedRef` synchronous boolean flags
- Prevents duplicate speech recognition during user retry attempts
- Same pattern as existing MathLesson1

**Audio Flow**
- Uses Web Speech API: `SpeechRecognition` for voice input, `SpeechSynthesis` for Mrs. Love output
- `createSpeechRecognizer()` factory handles all recognition setup
- `speakText()` / `stopSpeaking()` abstractions for consistent voice

**Word Matching**
- **Exact match** for sight words (normalized: lowercase, punctuation stripped)
- **70% overlap** for story sentences (allows pauses, stumbles, slight variations)
- Symmetric normalization: both transcript and target get same lowercase + punctuation strip

**Config Storage**
- `localStorage` key: `reading-config-${studentId}`
- Format: `{ sightWords: [...], phonicsWords: [...], story: "..." }`
- On lesson start, loads from localStorage; falls back to hardcoded defaults if nothing set

**Progress Persistence**
- Calls `saveProgress()` at lesson completion (same KV sync as math lessons)
- Saves: `lessonType: 'reading'`, phase scores, timestamp, studentId
- Visible in parent dashboard immediately (multi-device sync for shared links)

## User Experience Flow

1. Kiara taps **📖 Start Reading Lesson** on home screen
2. Reading Lesson page loads → Mrs. Love auto-speaks intro ("Hello, Kiara! Let's practice...")
3. Kiara taps **🎤 Read it!** for each sight word
   - Match → random praise spoken, advance
   - No match → "Look closely — try again!" (up to 2 attempts, then reveal + move on)
4. Phase transition → Mrs. Love speaks motivational line, phonics words begin (spaced letters)
5. Same matching/hint/retry flow for phonics (still 5 words)
6. Phase transition → Short story revealed sentence-by-sentence (active = yellow, upcoming = dimmed)
7. Kiara taps **🎤 Read this sentence!** per sentence
   - ≥70% word overlap → praise, advance
   - <70% overlap → encouraging comment + auto-advance
8. Final sentence → completion screen with score breakdown, celebration emoji, percentage + star badge
9. Tap **← Back to Home** to return

## Fixes Applied During Development

| Issue | Root Cause | Fix |
|-------|-----------|-----|
| Story split only on `.` | Parser used simple `.split('.')` | Changed to `split(/[.!?]+/)` |
| Asymmetric transcript normalization | Transcript stripped but targetWord wasn't | Both now normalize: `toLowerCase().replace(/[^a-z\s]/g, '')` |
| Microphone stays open on back-tap | No unmount cleanup | Added `useEffect` return with `recognizer.stop() + stopSpeaking()` |
| Back button touch target too small | Only 32px tall | Added `min-height: 44px; display: flex; align-items: center` |
| No responsive styling | Small phones broke layout | Added `@media (max-width: 400px)` reducing word font to 2.5rem |
| Reading sessions appear twice in dashboard | Dashboard listed all sessions + filtered reading sessions | Changed map from `sessions` to `mathSessions` |
